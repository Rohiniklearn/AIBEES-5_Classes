# Enterprise RAG Pipeline — GCP Vertex AI

Script-based end-to-end RAG pipeline.
Runs entirely within GCP Vertex AI Workbench or any authenticated Python environment.

---

## Prerequisites
- Create virtual environment for this project
- APIs enabled: `aiplatform.googleapis.com`, `storage.googleapis.com`
- Authenticated: `gcloud auth application-default login`
- Python 3.10+


## File Layout

## .env file content
GOOGLE_API_KEY=Your Key
GCP_PROJECT_ID=Your ProjectID
GCP_REGION=us-central1

## Project Folder Structure
```
rag_pipeline/
├── config.py                 # Central config — edit this first
|-- .env                      # Place your .env file here with api keys
├── step_01_setup_gcs.py      # Create GCS buckets
├── step_02_create_index.py   # Create & deploy Vector Search index
├── step_03_ingest.py         # Ingest PDFs → embed → upsert
├── step_04_query.py          # Interactive / single-shot RAG query
├── step_05_cleanup.py        # Delete all GCP resources
├── requirements.txt
└── .rag_config.json          # Auto-generated by step_02 (index/endpoint IDs)
```





## Run Order

### 1. Edit config
Open `config.py` and set your `PROJECT_ID`, bucket names, and region.

---

### 2. Create GCS buckets
```bash
python step_01_setup_gcs.py
```
Creates `SOURCE_BUCKET` (for raw PDFs) and `EMBED_BUCKET` (for embeddings).

---

### 3. Upload your PDFs
Upload file from UI and monitor the progress in Google

or 

```bash
gsutil cp /local/path/*.pdf gs://<SOURCE_BUCKET>/pdfs/
```
## OR
# you can configure the required pdfs from web and load them to the GCS bucket directly using the script

chmod +x download_medical_pdfs.sh ./download_medical_pdfs.sh

bash download_medical_pdfs.sh

---

### 4. Create & deploy Vector Search index
```bash
python step_02_create_index.py
```
⚠ Takes ~30–45 minutes. Run once. Saves `.rag_config.json` locally.

---

### 5. Ingest documents
```bash
# First-time: ingest all PDFs
python step_03_ingest.py --mode full

# Add new PDFs later (skips already-ingested files)
python step_03_ingest.py --mode incremental

# Ingest from a specific subfolder
python step_03_ingest.py --mode incremental --prefix pdfs/new/
```

---

### 6. Query
```bash
# Interactive REPL
python step_04_query.py

# Single question
python step_04_query.py --question "What is this document  about?"

# Show retrieved source chunks
python step_04_query.py --question "What is the refund process?" --verbose

# Retrieve more chunks
python step_04_query.py --top-k 10
```

---

### 7. Cleanup (save credits)
```bash
# Interactive — prompts before each deletion
python step_05_cleanup.py

# Non-interactive (CI / automation)
python step_05_cleanup.py --yes

# Also delete the raw PDF source bucket
python step_05_cleanup.py --yes --delete-source
```

## Key Design Decisions

| Decision | Detail |
|---|---|
| Embedding model | `gemini-embedding-001` (3072 dims)
| Index update mode | `STREAM_UPDATE` — incremental upserts, no full rebuild needed |
| Distance metric | `DOT_PRODUCT_DISTANCE` (= cosine for normalised vectors) |
| Incremental tracking | `metadata/ingested_files.json` in GCS tracks processed files |
| LLM | `gemini-2.5-pro` or `gemini-2.5-flash` via `ChatVertexAI` |

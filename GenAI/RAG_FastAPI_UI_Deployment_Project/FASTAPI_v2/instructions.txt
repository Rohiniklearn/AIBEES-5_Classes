## To know the workbench account you are using for deployment
curl -s "http://metadata.google.internal/computeMetadata/v1/instance/service-accounts/default/email" \
  -H "Metadata-Flavor: Google"
  
Sample output : 633943311262-compute@developer.gserviceaccount.com 

## Check if this exact account is reflecting in IAM & Admin -> IAM page
    - search for IAM in search bar and navigate to IAM & Admin -> IAM 
    - If Yes, add the missing role
	- If No, add the principle account with the sample output as reference and add the role 
	    - View by Principles -> Grant Access -> Add Principals (add 633943311262-compute@developer.gserviceaccount.com as a name) and    
		  add the required role (based on the error you are getting while doing the deployment)
	- Retry your deployments
